Updates!

Fixed some issues which allowed users to post HTML into page2 (Guestbook). Now it just removes any '<' or '>' which pretty much breaks any HTML.

I've been working on various improvements to the server code. Up until now if a client tried to send a .XML to my server, it'd just crash outright. That /should/ be fixed now.

Note: Haven't made a 404 page so far so if you do something the server doesn't like, it 404's then gives you the home page. Didn't want to flashbang anyone with a stark white 404 screen.

Fixed JSON submission. If the wrong keys are used or no keys are present, it'll just sign with 'not provided' in the missing/incorrect fields

Added some calles to a Weather API. Can put in your city, zip, state and get some current weather info back. It's currently really ugly but I'll clean it up eventually.

Added more API calls. This time the calls are handled by scripts imported by the HTML instead of the server doing it all. It doesn't really run any differently, but mistakes don't crash the server and instead the script crash but the rest of the page keeps working, so that's pretty nice. The Animal pictures are handled with a GET call to a server that returns an object with a image URL. The script appends the URL to the IMG tag already in the HTML. I could have just had the script create the IMG tag on demand, and append it to the HTML but there would have been extra logic required to update a possibly existing IMG tag or it'd just keep appending more and more images to the bottom of the page. That might be cool, but you could just keep clicking and end up with an infinite number of pictures on one page and would require scrolling to the bottom to see the new pictures. Could add logic to append the img to the front, but nah, that would've been more annoying.
